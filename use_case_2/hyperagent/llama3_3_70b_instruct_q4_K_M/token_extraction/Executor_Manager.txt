The error message you're seeing indicates that the Tkinter library, which is Python's de-facto standard GUI (Graphical User Interface) package, is trying to create a window but cannot find a display to render it on. This is because the `$DISPLAY` environment variable is not set.

When you run a graphical application in a non-graphical environment (like a remote server or a Docker container without a graphical interface), you need to tell the application where to send the graphical output. The `$DISPLAY` variable does this by specifying the display server that the application should use.

Here are a few potential solutions:

### 1. Set the DISPLAY Environment Variable

If you're running your script on a local machine with a graphical interface, you can try setting the `DISPLAY` environment variable before running your Python script:

```bash
export DISPLAY=:0
python login_page.py
```

Replace `:0` with the appropriate display number if necessary.

### 2. Use Xvfb (Virtual Frame Buffer)

If you're on a Linux system and want to run your GUI application without actually displaying it, you can use `Xvfb`, which is a virtual framebuffer for X:

1. Install `xvfb`:
   ```bash
   sudo apt-get update
   sudo apt-get install xvfb
   ```

2. Run your script with `Xvfb`:
   ```bash
   Xvfb :1 -screen 0 1024x768x24 &
   export DISPLAY=:1
   python login_page.py
   ```

This will start a virtual display and then run your Python script using that display.

### 3. Use a Headless GUI Testing Tool

If you're trying to automate tests for your GUI application, consider using tools designed for headless GUI testing, such as `PyVirtualDisplay` or `Selenium` with a headless browser setup.

- **PyVirtualDisplay**: Allows you to create a virtual display for your GUI tests.
  ```python
  from pyvirtualdisplay import Display
  display = Display(visible=0, size=(1024, 768))
  display.start()
  # Your Tkinter application code here
  display.stop()
  ```

- **Selenium with Headless Browser**: Useful if you're testing web applications. You can configure browsers like Chrome or Firefox to run headlessly.

### 4. Modify Your Application

import os
